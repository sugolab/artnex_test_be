# ArtNex 제조업 AI 마케팅 자동화 SaaS 플랫폼 MVP 개발 문서

본 문서는 ArtNex 플랫폼의 PRD, ERD, 기능명세서, 개발명세서, WBS를 기반으로 재구성된 전체 내용입니다. 제조업 중심의 AI 마케팅 자동화 기능을 중심으로 프로젝트 개요, 일별 작업 스케줄, 주요 가정 및 리스크 관리 등을 마크다운 형태로 제공합니다. 개발 기간은 2025-10-23 ~ 2026-01-16 (총 58 영업일, 개발 49 영업일)로 설정되며, 프론트엔드 3인, 백엔드 1인 병렬 개발을 가정합니다.

## 1. 프로젝트 개요

### 1.1 프로젝트 요약 (PRD 및 개발명세서 기반)
- **제품명**: ArtNex (아트넥스)
- **유형**: AI 기반 브랜드·디자인·마케팅 통합 SaaS 플랫폼 (제조업 중심 AI 마케팅 자동화)
- **개발 기간**: 2025.10.23 ~ 2026.01.16 (총 58영업일, 개발 49영업일)
- **개발 인력**: Frontend 3명 (Vue + Tailwind) / Backend 1명 (Python FastAPI)
- **핵심 목표**: 브랜드 진단 → 디자인 생성 → 마케팅 실행 → 리포트 분석 AI 자동화 원스톱 플랫폼 구축. 제조 기업의 마케팅 ROI 향상, 시장 경쟁력 강화, 사용자(마케터/디자이너/경영진) 시간 절감.
- **성공 지표 (KPI)**: 사용자 유지율 70% 이상, 월간 활성 사용자(MAU) 1,000명, AI 리포트 생성 시간 5분 이내.

### 1.2 목적 및 목표 (PRD 기반)
- **목적**:
  1. 제조업 브랜드의 마케팅 프로세스를 AI로 자동화하여 시간과 비용을 절감.
  2. 브랜드 진단, 콘텐츠 생성, 캠페인 관리, 성과 분석을 하나의 플랫폼에서 통합.
  3. 사용자(제조 기업 마케터, 디자이너)가 쉽게 접근할 수 있는 직관적 인터페이스 제공.
- **목표**:
  - 비즈니스 목표: 제조 기업의 마케팅 ROI 향상, 시장 경쟁력 강화 (사용자 유지율 70% 이상).
  - 사용자 목표: 브랜드 등록부터 리포트 출력까지 원스톱 워크플로우 제공 (MAU 1,000명).
  - 기술 목표: AI 기반 리포트 생성 시간 단축 (5분 이내).

### 1.5 시스템 아키텍처 (개발명세서 기반)
ArtNex는 다계층 분산 아키텍처를 채택하여 안정성과 확장성을 확보합니다.

- **Frontend**: Vue3 + Tailwind CSS 기반의 SPA(Single Page Application)로 개발. 주요 뷰: Dashboard, Brand, Insight, Report, Design, Shortform, Campaign.
- **Backend**: FastAPI (Python)를 사용하여 고성능 RESTful API 서버 구축. 모든 비즈니스 로직과 AI 분석 모듈 통합 담당.
- **Database**: PostgreSQL 메인 데이터 저장소. 브랜드, 프로젝트, 계약, 리포트, 캠페인 관련 데이터 구조화.
- **Infra & AI**: AWS 환경 운영.
  - AWS EC2: 메인 애플리케이션 서버 호스팅.
  - AWS S3: 사용자 생성 파일 (계약서, 리포트 PDF, 디자인 결과물 등) 저장.
  - AWS Lambda: AI 연산 모듈 (GPT API, Ideogram API 호출 및 후처리) 처리. 서버 부하 분산, 비용 효율성 높임.

## 3. 주요 가정 및 리스크 관리

### 3.1 주요 가정 (PRD, 개발명세서 기반)
- **영업일 계산**: 주말 및 공휴일(한글날 2025-10-09, 크리스마스 2025-12-25, 신정 2026-01-01) 제외, 총 58 영업일 기준 (버퍼 포함).
- **병렬 작업**: 개발자 간 의존성 최소화 (e.g., ERD 테이블 공유), AI 연동 지연 시 Lambda 스케일링.
- **성공 지표**: PRD KPI 준수 (AI 리포트 생성 5분 이내, MAU 1,000명, 사용자 유지율 70%).
- **기술 가정**: AWS 환경에서 운영, PostgreSQL 정규화 원칙 준수, 외래 키 제약 ON DELETE CASCADE 또는 RESTRICT 정책 적용.

### 3.2 리스크 관리 (PRD, 개발명세서 기반)
- **AI 응답 지연**: 목표 5초 이내. 대책: Mock 테스트, Lambda 스케일링, 비동기 처리.
- **보안 리스크**: 데이터 암호화, JWT 인증, RBAC 적용 강화. 민감 데이터 DB 암호화.
- **확장성 리스크**: 외부 서비스 연동 (SNS API, 이미지 API). 대책: API 게이트웨이 패턴, 느슨한 결합.
- **유지보수성**: 모듈화된 Vue 컴포넌트, API Layer 분리, 단위 테스트 작성.
- **기타 리스크**: 기능명세서 준수 (PDF Export 다국어 지원), 공휴일 영향 고려. 리스크 발생 시 버퍼 기간 활용.

### 공통 기능
- AI 자동화: 입력 데이터 기반 자동 진단·추천·생성.
- 파일 입출력: 이미지·PDF·CSV 업로드/다운로드 (AWS S3 연동).
- 권한 관리: 관리자/일반/뷰어 (RBAC, JWT 기반).
- 데이터 시각화: KPI·트렌드 차트 (Chart.js 기반).
- PDF Export: 리포트, 디자인, 캠페인 리포트 출력 (ReportLab 기반, 다국어/워터마크 지원).
- UI 공통요소: BreadCrumb, ProgressBar, 카드형 리스트 (Vue 컴포넌트화).

### 사용자 워크플로우
1. 로그인 → 메인 대시보드 진입.
2. 브랜드 등록 → 진단 (5단계 폼 + AI 분석).
3. 디자인 스튜디오 → 브랜드 톤앤매너 기반 시안 생성.
4. 숏폼 스튜디오 → 영상 스토리보드 자동 생성.
5. 콘텐츠 트래킹 → 키워드 기반 트렌드 분석.
6. 캠페인 매니저 → 광고 캠페인 생성 및 관리.
7. 캠페인 리포트 → 성과 데이터 분석 + PDF 출력.
